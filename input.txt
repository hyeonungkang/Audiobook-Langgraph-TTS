# 🧠 방산 취업, 이렇게 임하세요
## 태도·생각정리·팁 (마인드셋 중심)

## 📍 Part 1: 근본적 태도의 재설정

### 1. "내가 해야 할 일"을 명확히 하기

**❌ 취준생들의 흔한 착각**
```
"합격이 목표다"
"면접에서 좋은 인상을 주면 된다"
"자소서는 템플릿만 따르면 된다"
```

**✅ 올바른 마인드셋**
```
"나는 이미 충분히 준비된 사람인가?"
"내 경험이 진짜 가치 있는 것인가?"
"면접관이 '이 사람은 진짜 했겠네'라고 느낄까?"

→ 합격은 그 다음이다
```

**실제 의미**
```
합격 = 나라는 사람이 온전히 전해졌을 때의 결과

팁으로 어떻게 할 수 없는 부분:
- 자신감 (경험이 뒷받침해야 함)
- 진심 (만들 수 없음)
- 깊이 (이해하지 않으면 들통남)

따라서 먼저 준비하고, 그 다음 어떻게 전할지 생각
```

---

### 2. "이것은 합격을 위한 게임이 아니다"

**❌ 흔한 접근**
```
"면접관들이 좋아할 만한 대답을 준비해야지"
"자소서 템플릿을 보고 따라 써야지"
"최대한 좋게 포장해야지"
```

**✅ 올바른 접근**
```
"내 진짜 경험은 뭐지?"
"그 경험에서 진짜 배운 게 뭐지?"
"내가 진짜 도움이 될 사람인가?"

→ 그것을 솔직하게 전하기
```

**왜 이게 중요한가?**
```
사실이 아닌 대답은 5초 안에 들통난다

면접관은:
- 당신과 같은 기술을 하던 사람
- 수백 명의 지원자를 봐온 사람
- 거짓을 구분하는 데 매우 능숙한 사람

"연기"는 절대 통하지 않는다
"솔직함"이 최강의 전략이다
```

---

### 3. "나는 남과 비교하지 않는다"

**❌ 취준의 함정**
```
"저 사람은 대기업 인턴을 했는데..."
"저 사람은 논문을 냈는데..."
"저 사람은 학점이 높은데..."

→ 자신감이 떨어지고, 준비가 흐트러짐
```

**✅ 올바른 마인드셋**
```
"나는 나의 경험을 최대한 깊이 있게 준비한다"
"다른 사람의 수치가 아니라 나의 깊이를 본다"
"면접관이 궁금해할 것들을 미리 생각해본다"

→ 경쟁이 아니라 준비
```

**구체적 예**
```
❌ "나는 인턴을 못 했으니 떨어질 거야"

✅ "나는 팀 프로젝트에서 뭘 배웠지?
   그것을 인턴만큼 깊이 있게 설명할 수 있나?
   면접관의 질문에 구체적으로 답할 수 있나?"

→ 없는 경험을 탓할 게 아니라
   있는 경험을 극대화하기
```

## 📍 Part 2: 생각을 정리하는 방법

### 1. "경험의 구조화" (가장 중요)

**단계별 정리법**

```
모든 경험에는 구조가 있다:

[1단계] 상황 (Situation)
└─ "언제, 어디서, 뭘 하고 있었나?"
   예: "3학년 1학기 열전달 프로젝트에서..."

[2단계] 문제 (Problem)
└─ "뭐가 문제였나?"
   예: "처음에는 시뮬레이션 결과가 이론과 안 맞았어"

[3단계] 행동 (Action)
└─ "내가 뭘 했나?" (구체적으로)
   예: "논문을 3개 읽었고,
        교수님께 물어봤고,
        수치해석을 손으로 다시 계산해봤어"

[4단계] 결과 (Result)
└─ "그래서 뭐가 됐나?" (구체적 수치)
   예: "시뮬레이션과 이론의 오차가 3%로 줄었어"

[5단계] 배운 점 (Learning)
└─ "이게 나에게 뭘 의미하나?"
   예: "엔지니어는 겸손해야 한다는 것을 배웠어.
        데이터와 이론이 맞지 않으면,
        내 계산을 먼저 의심해야 한다"
```

**왜 이게 중요한가?**
```
이 구조가 있으면:

1) 자소서에서 글을 어떻게 쓸지 명확해짐
2) 면접관이 질문할 때 답하기 쉬움
3) 자신감이 생김 (준비되었으니까)
4) 거짓말이 불가능 (구체적이니까)
```

**실제 적용**
```
프로젝트 3개, 각각 이 구조로 정리

1번 프로젝트: ANSYS 해석
   [S] 3학년 열전달 프로젝트
   [P] 시뮬레이션 결과 vs 이론 불일치
   [A] 논문 읽기, 교수 상담, 손 계산
   [R] 오차 3% 달성
   [L] 겸손함, 기초의 중요성

2번 프로젝트: LSTM 모델
   [S] 4학년 캡스톤 설계
   [P] 데이터 부족 (1000개만 있음)
   [A] 신호처리로 특성 추출, 전이학습 적용
   [R] 정확도 92% 달성
   [L] 창의적 문제 해결, 깊이

3번 프로젝트: 팀 프로젝트
   [S] 경진대회 3개월
   [P] 팀원들의 주도성 부족
   [A] 역할 분담, 주간 미팅, 동기부여
   [R] 우상 수상
   [L] 리더십, 책임감

→ 이제 자소서도 쓸 수 있고, 
   면접도 볼 수 있고,
   자신감도 있다
```

### 2. "약점을 장점으로 변환하기"

**가장 실용적인 팁**

```
❌ 약점을 숨기려고 하면 들통남
   → "인턴을 못 했어요" 하고 끝내면 떨어짐

✅ 약점을 설명으로 바꾸면 가점
   → "인턴 대신 캡스톤 설계에서
      LSTM으로 RUL 예측 모델을 만들었고,
      92% 정확도를 달성했습니다"
```

**구체적 변환 예시**

```
 학점이 낮다면?
❌ "죄송합니다. 학점이 낮아요"
✅ "학점보다는 실전 경험을 중시했습니다.
   특히 ANSYS로 구조해석을 깊이 있게 배웠고,
   [구체적 프로젝트]를 통해 
   실무 역량을 키웠습니다"

 인턴 경험이 없다면?
❌ "인턴을 못 했어요"
✅ "인턴 대신 [자체 프로젝트 3개]로
   기계공학 + AI 융합 경험을 쌓았습니다.
   특히 신호처리와 LSTM의 조합으로
   데이터 부족 환경에서도 
   92% 정확도를 달성했습니다"

 대기업 경력이 없다면?
❌ "대기업에서 일해본 적이 없어요"
✅ "스타트업 프로젝트와 학교 연구를 통해
   빠른 배움과 책임감을 경험했습니다.
   [구체적 사례]에서 주인의식으로
   프로젝트를 주도했습니다"
```

**원리**
```
방산은 당신의 결핍을 보는 게 아니라
당신이 있는 것으로 뭘 만들었는가를 본다
```

### 3. "질문을 역으로 생각하기"

**가장 강력한 준비법**

```
자소서를 쓸 때:
"이 항목을 읽은 면접관이 뭘 물어볼까?"
를 미리 생각하고 쓴다

예를 들어:

자소서: "ANSYS로 구조해석을 했고, 
         안전계수 1.52를 달성했습니다"

면접관이 물을 질문들:
- "Mesh 사이즈는 어떻게 정했나?"
- "왜 이 재료를 선택했나?"
- "만약 안전계수가 1.5 미만이었다면?"
- "이론과 시뮬레이션이 다르면 어떻게 하나?"
- "다른 팀원은 뭘 했나?"

→ 이 질문들에 미리 답을 준비해두고
   자소서를 쓴다
```

**구체적 방법**

```
Step 1: 자소서 항목을 읽는다
Step 2: "면접관이라면 뭘 물을까?" 
        예상 질문 5개를 만든다
Step 3: 각 질문에 구체적 답을 준비한다
Step 4: 그 답들이 자소서에 나올 수 있도록 
        자소서를 수정한다

결과: 
- 자소서는 미리 면접 대비를 한 것
- 면접에서 예상 질문이 나오면 완벽히 대답
- 예상 못한 질문이 나와도 깊이가 있어서 대응 가능
```

## 📍 Part 3: 구체적 팁들 (실행 단계)

### Tip 1: "구체성의 법칙"

**절대 지켜야 할 원칙**

```
❌ "좋은 결과를 얻었습니다"
✅ "응력이 250 MPa에서 248 MPa로 감소했습니다"

❌ "프로젝트를 성공적으로 완료했습니다"
✅ "5주 안에 완료했고, 교수 평가는 A+였습니다"

❌ "열심히 했습니다"
✅ "논문 10개를 읽었고, 교수 5명과 상담했습니다"
```

**왜?**
```
구체성 = 진실의 증거
추상성 = 거짓일 가능성

면접관은 구체성을 통해 당신이 
"정말 했는지" "얼마나 깊이 있게 했는지"
판단한다
```

---

### Tip 2: "숫자로 말하기"

**강력한 표현법**

```
모든 설명에 숫자를 붙인다:

"설계했습니다" (약함)
→ "두께를 3mm에서 4mm로 늘려 
   안전계수를 1.12에서 1.52로 높였습니다" (강함)

"정확도가 높습니다" (약함)
→ "정확도 92.3%를 달성했습니다" (강함)

"빨리 끝냈습니다" (약함)
→ "예정된 6주에서 5주만에 완료했습니다" (강함)
```

**효과**
```
숫자 = 신뢰도 50% 향상
```

---

### Tip 3: "깊이와 넓이의 균형"

**자주 하는 실수**

```
❌ 깊이만 있는 경우
"ANSYS를 6개월간 공부했고, 
 Mesh 생성부터 후처리까지 완벽합니다"
→ "그래서 뭘 만들었는데?"

❌ 넓이만 있는 경우
"ANSYS, MATLAB, Python, CAD 다 할 수 있습니다"
→ "어느 정도 수준인데?"
```

**✅ 올바른 균형**
```
주력 기술 (깊이):
"ANSYS로 구조해석을 했고,
 Mesh부터 후처리까지 실제 프로젝트에서
 안전계수 1.52를 달성했습니다"

보조 기술 (넓이):
"추가로 MATLAB로 수치계산을 하고,
 Python으로 데이터를 처리했습니다"

→ 깊이 있으면서도 다양한 기술을 사용
```

### Tip 4: "거짓말은 하지 않고, 솔직함의 가치"

**가장 중요한 가치관**

```
"Mesh 사이즈는 어떻게 정했나요?"

❌ 틀린 대답:
"최적 값 계산 공식으로 정했습니다"
(면접관: "그런 공식은 없는데?")
→ 떨어짐

✅ 올바른 대답:
"처음에는 대략 5mm로 했는데,
 교수님 조언으로 수렴성 분석을 했습니다.
 3mm와 1mm를 시도해보니 차이가 거의 없어서
 계산 속도를 고려해 3mm로 결정했습니다"
→ 깊이, 신뢰도, 판단력이 모두 보임
```

**왜 솔직함이 최고의 전략인가?**
```
1) 거짓은 항상 들통난다
   → 팔로우업 질문 하나에 무너짐

2) 솔직함은 자신감을 만든다
   → 면접관이 느낀다

3) 솔직함이 차별화다
   → 많은 지원자가 거짓말을 하니까,
      솔직한 사람이 더 돋보임
```

---

### Tip 5: "회사 분석의 깊이"

**차별화되는 취준생의 비결**

```
일반 취준생: 회사 홈페이지 읽기
→ "귀사는 좋은 회사입니다"

합격하는 취준생: 
- 지난 2년 사보 읽기
- 최근 뉴스 10개 이상 읽기
- 제품 분석 (기술적으로)
- 경쟁사와 비교
- 향후 전략 예측
→ "한화의 항공사업은 22.7%인데,
   KF-21 국산화와 누리호로 이게 올라갈 것 같습니다.
   이 성장에 엔진 설계로 기여하고 싶습니다"
```

**효과**
```
"아, 이 사람은 우리를 진짜 알겠네"
→ 자소서 합격률 70% → 90% 증가
```

---

### Tip 6: "자신감과 겸손의 균형"

**가장 어려운 부분**

```
❌ 과신
"저는 ANSYS를 완벽히 합니다"
(면접관의 심화 질문에 답 못함)

❌ 과겸손
"저는 아직 많이 부족합니다"
(자신감이 없어 보임)

✅ 올바른 태도
"저는 ANSYS로 구조해석 프로젝트를 
 3개 완료했고, 깊이 있게 할 수 있습니다.
 
 다만 CFD 같은 분야는 아직 배워야 할 부분이 있고,
 입사 후 더 깊이 있게 배우고 싶습니다"
→ 자신감 + 겸손 + 배우려는 자세
```

### Tip 7: "기록하기"

**구체적 실행 방법**

```
모든 프로젝트를 기록하라:

[프로젝트명]: ANSYS 구조해석
[기간]: 2024.03.01 ~ 2024.05.31 (3개월)
[팀원]: 5명 (내 역할: Mesh + 해석)

[목표]: 안전계수 1.5 이상 달성

[실행 과정]:
Day 1-7: CAD 모델 확보 및 분석
Day 8-14: Mesh 생성 (5mm, 3mm, 1mm 수렴성)
Day 15-21: 해석 수행 및 결과 분석
→ 최대 응력 250 MPa, 안전계수 1.12 (부족)
Day 22-35: 설계 최적화
→ 두께 3mm→4mm, 안전계수 1.52 달성
Day 36-42: 보고서 작성

[결과]: 
- 안전계수 1.52 (목표 1.5)
- 무게 증가 8% (허용 10%)
- 교수 평가: A+

[배운 점]:
- 메시 수렴성 분석의 중요성
- 트레이드오프 최적화
- 이론과 시뮬레이션의 차이 이해

[자소서에서 사용]: 직무역량 항목
[면접 예상 질문]: 
- Mesh 사이즈 결정 방법?
- 왜 이 재료를 선택했나?
- 만약 안전계수 1.5 미만이었다면?
```

**효과**
```
- 자소서 쓸 때 구체적 근거 제시 가능
- 면접에서 흐릿한 대답 불가능
- 자신감이 생김
```

## 📍 Part 4: 마음가짐 (가장 중요)

### 1. "합격이 아니라 준비에 집중"

```
❌ 잘못된 목표 설정
"6개월 안에 무조건 합격해야 해"
→ 초조함, 급함, 깊이 부족

✅ 올바른 목표 설정
"6개월간 최고의 엔지니어가 되자"
→ 자신감, 깊이, 진심

→ 역설적이지만, 
   합격에 집중할수록 떨어지고,
   준비에 집중할수록 합격한다
```

### 2. "기준을 외부에서 내부로"

```
❌ 외부 기준
"다른 사람들은 뭘 하지?"
"저 사람이 합격했는데, 나는?"
"면접관이 뭘 원할까?"

✅ 내부 기준
"내가 할 수 있는 최선이 뭐지?"
"내 경험에서 배운 게 뭐지?"
"나는 이 회사에 뭘 기여할 수 있나?"
```

### 3. "실패를 배움으로 보기"

```
❌ 심리적 함정
서류가 떨어짐
→ "나는 부족한 사람이야"

✅ 올바른 시각
서류가 떨어짐
→ "뭐가 부족했나?"
→ "자소서를 더 깊이 있게 쓸 수 있겠다"
→ "회사 분석을 더 해야겠다"
→ 다음 원서에 반영

→ 떨어짐이 피드백이 된다
```

### 4. "당신은 충분하다"

```
자소서를 쓸 때, 면접을 볼 때, 
계속 느끼는 불안감:

"내가 충분할까?"
"다른 사람들이 더 잘하지 않을까?"

→ 이것은 준비가 부족해서가 아니라
   누구나 느끼는 감정이다

→ "나는 충분히 준비했다"
   "나의 경험은 진짜다"
   "내가 할 수 있는 최선을 했다"

→ 이 확신이 면접에서 빛난다
```

## 📍 최종 체크: 당신의 준비 상태

```
자신에게 묻기:

□ 나의 프로젝트 3개를 5분씩 설명할 수 있나?
  (자신 있게, 거짓 없이)

□ 내 경험에서 배운 게 뭔지 명확한가?

□ 회사를 진짜 알고 있나?
  (사보 2년, 뉴스 10개, 제품 분석)

□ 면접관이 물을 질문 10개에 답할 수 있나?

□ 내 약점을 장점으로 설명할 수 있나?

□ 솔직함을 유지하고 있나?
  (거짓말 없이, 자신감 있게)

□ 다른 사람과 비교하지 않고 있나?

□ "나는 충분하다"고 느껴지나?

→ 모두 YES라면, 당신은 준비된 것이다
```

## 🎯 결론: 3가지만 기억하세요

```
내부에 집중
   = 합격 전략이 아니라 준비에 집중
   = 외부 비교가 아니라 내 기준 세우기

깊이 > 폭
= 많은 기술보다 깊이 있는 3개
 = 구체성 (숫자, 사례)이 신뢰를 만든다

 솔직함이 최강
   = 거짓은 들통난다
   = 솔직함이 가장 강한 차별화
   = 당신의 진짜 경험이 가장 매력적이다
```
